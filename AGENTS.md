# 599549.com 开发规则

## 1. 项目身份

本项目是 599549.com 二次开发维护项目。

系统结构：

- 澳门前台
- 香港前台
- 共用后台
- PHP 原生项目
- 前后端不分离
- MySQL / MariaDB 数据库

本项目不是新项目，不允许推翻重写。


## 2. 工作方式

每次开发前必须先确认：

- 当前目录
- Git 状态
- Git remote
- 项目结构
- 本次涉及文件
- 澳门 / 香港影响
- 后台影响
- 数据库 / 接口影响

必须先执行：

git status --short

如果工作区不干净，先说明已有改动，不得覆盖用户未提交内容。


## 3. 开发单位

本项目禁止碎片式开发。

禁止：

- 只改一个按钮
- 只改一行 CSS
- 只修一个角落
- 只做澳门不看香港
- 只做前台不看后台
- 只写页面不接数据
- 只接数据不处理空状态

正确方式：

一个任务按一个模块闭环处理。

模块闭环包括：

- 前台页面
- 澳门 / 香港对应页面
- 后台管理
- API
- 数据读写
- UI 适配
- 空状态
- 错误状态
- 基础验证


## 4. 修改范围

允许一次修改多个文件，但必须属于同一个业务模块。

禁止：

- 跨模块顺手修改
- 顺手重构
- 顺手优化
- 批量格式化全项目
- 批量替换未知范围代码
- 删除未确认文件
- 清空日志 / 缓存 / 上传目录


## 5. 澳门 / 香港规则

澳门和香港必须同步考虑。

规则：

- 共用后台必须区分澳门 / 香港
- 数据必须使用现有 region / site / system 字段
- 不允许澳门数据覆盖香港数据
- 不允许香港数据覆盖澳门数据
- 不允许只改澳门漏掉香港
- 不允许只改香港漏掉澳门
- 前台结构应保持一致，允许地区文案不同

允许不同：

- 系统名称
- 品牌标识
- 地区文案
- 开奖时间
- 开奖期号
- 开奖结果
- 开奖数据来源
- 开奖历史


## 6. 开奖规则

开奖逻辑属于高风险区域。

禁止：

- 合并澳门 / 香港开奖数据
- 共用澳门 / 香港开奖时间
- 共用澳门 / 香港开奖期号
- 无关任务修改开奖逻辑
- 后台共用导致开奖数据串区

涉及开奖时必须说明：

- 是否影响澳门开奖
- 是否影响香港开奖
- 是否影响开奖时间
- 是否影响开奖期号
- 是否影响开奖结果
- 是否影响开奖数据来源


## 7. UI 开发规则

UI 修改必须基于：

- 现有 HTML 结构
- 现有 CSS
- 同类页面
- 用户截图

禁止：

- 自由设计
- 重做页面结构
- 改变模块顺序
- 改变信息层级
- 随意改全局 CSS
- 大量使用 !important
- 澳门一套 UI、香港一套 UI
- 用固定高度硬压动态内容
- 图片拉伸变形

允许：

- 修错位
- 修溢出
- 修间距
- 修对齐
- 修图片比例
- 修手机端显示异常

移动端优先：

- 页面不能横向滚动
- 内容不能被底部导航遮挡
- 按钮点击区域不能过小
- 卡片不能压缩变形
- 长标题必须换行或省略


## 8. CSS / HTML 治理

CSS 修改优先级：

1. 组件 CSS
2. 模块 CSS
3. 页面 CSS
4. 全局 CSS

禁止优先改全局 CSS，除非问题确实来自全局样式。

HTML 禁止：

- 无意义 div 嵌套
- 复制大量重复结构
- 为一个页面问题制造新结构
- 用 CSS 掩盖 HTML 结构错误


## 9. 后台开发规则

后台是共用后台，必须服务澳门和香港。

后台模块必须考虑：

- 列表
- 分页
- 筛选
- 新增
- 编辑
- 删除 / 状态切换
- region 写入
- 权限检查
- 登录不受影响

禁止：

- 重写后台架构
- 修改登录系统
- 修改权限体系
- 绕过现有接口方式
- 绕过现有数据库封装方式


## 10. 数据库和接口规则

默认禁止：

- 修改数据库结构
- 修改字段含义
- 修改接口地址
- 修改接口请求方式
- 修改接口返回结构
- 修改登录逻辑
- 修改权限逻辑
- 修改开奖数据逻辑

允许只读审计：

- 查看表结构
- 查看字段
- 查看索引
- 查看少量样本数据
- 查看 region / site / system 字段

如果必须修改数据库或接口，先输出方案，等待确认。


## 11. 安全规则

所有用户输入默认不可信。

必须防范：

- SQL 注入
- XSS
- CSRF
- 越权访问
- 文件上传风险
- 上传目录执行 PHP
- 敏感信息泄露
- 澳门 / 香港数据串区
- 错误信息暴露路径、SQL、密钥、token

禁止开发：

- 下注
- 赔率
- 彩票销售
- 购彩
- 提现
- 返奖
- 资金结算
- 赌博交易
- 赌博资金流转
- 非法引流


## 12. 性能规则

用户说慢、卡、加载久时，不能直接改 UI。

必须先查：

- 404
- 500
- PHP Fatal
- 日志刷屏
- API 重复请求
- JS 轮询过快
- 慢查询
- CSS / JS 重复加载
- 后台列表无分页
- CPU / 内存 / 磁盘异常

没有证据，不做性能优化。


## 13. 禁止命令

未经明确授权，禁止执行：

git reset
git clean
rm -rf
删除上传目录
清空数据库
清空日志
清空缓存
重启服务器
修改 stable 标签


## 14. 完成标准

任务完成必须满足当前范围内的完成标准。

前台：

- 页面能打开
- 数据能显示
- 空状态正常
- 手机端无横向溢出
- PC 端无明显错位
- 澳门 / 香港结构一致
- 不影响顶部、底部、开奖卡片、公告区

后台：

- 页面能打开
- 列表正常
- 分页正常
- 筛选正常
- 新增 / 编辑 / 状态切换按范围验证
- 澳门 / 香港数据不串区
- 权限未绕过
- 登录未受影响

API：

- 请求正常
- 返回结构兼容
- 错误处理明确
- 不泄露敏感信息
- region 隔离正确


## 15. 输出格式

每次任务完成后按以下格式输出：

【本次模块】
【任务类型】
【已查看文件】
【修改文件】
【实际完成】
【澳门 / 香港 / 后台影响】
【数据库 / 接口 / 权限 / 登录 / 开奖影响】
【UI检查】
【验证结果】
【风险说明】
【Git状态】
【下一步建议】


## 16. 核心纪律

不重构，只演进。
不乱猜，只看代码。
不碎改，只做模块闭环。
不自由设计，只维护现有体系。
不只做澳门漏香港。
不只做香港漏澳门。
不只做前台漏后台。
不改登录权限。
不乱碰开奖。
不编造结果。
不假装验证。
不扩大范围。
不破坏 stable。
