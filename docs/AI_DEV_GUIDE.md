# AI 开发推进指引

> 当前主用 AI 规则文件。旧纠偏类文件已归档，不再作为常驻规则读取。

## 1. 默认工作模式

项目处于半成品持续开发期。除非用户明确要求只检查、只审计、只排查不修改，默认进入开发推进模式。

默认目标：

1. 优先推进当前功能或页面完成。
2. 基于现有代码继续开发，不凭空新增并行业务系统。
3. 保持业务、接口、数据库、权限、登录逻辑不变。
4. 保持澳门 / 香港开奖时间、开奖结果、期号、数据来源独立。
5. 同类位置合理同步，但不无限扩大范围。
6. 已纠正的问题不重复审计，不把普通开发任务重新拉回旧纠偏模式。

## 2. 输出方式

普通开发、修复、优化完成后，简短输出：

```text
## 本次完成：

## 同步处理：

## 保持不变：

## 验证方法：
```

只有用户明确要求，或涉及根因、性能、数据库、接口、权限、登录、积分、购买授权、安装初始、澳门 / 香港开奖独立逻辑时，才补充详细说明。

## 3. 专项模式触发

只有用户明确出现以下意图时，才进入专项模式：

1. 说“纠偏、逻辑混乱、功能主线不清”时，进入功能逻辑纠偏。
2. 说“性能卡顿、慢加载、反应迟钝”时，进入性能纠偏。
3. 说“只检查不修改、只排查不修复、审计”时，进入只读审计。
4. 说“安全风险、权限风险、登录风险”时，进入对应安全检查。

专项模式结束后，回到开发推进优先，不长期停留在纠偏状态。

## 4. 业务保护

默认不改：

1. 接口地址、请求方式、参数名、返回结构。
2. 数据库表结构、字段含义、索引。
3. 登录、权限、CSRF、Session。
4. 积分余额、扣减、消费记录、购买授权链路。
5. 澳门 / 香港开奖时间、开奖结果、期号、数据来源。
6. 已确认正确的页面、样式、接口和业务链路。

如确需触碰以上内容，先说明原因、影响范围和验证方法，再等待用户确认。

## 5. 同类同步

同类位置需要合理同步：

1. 同一个字段来源。
2. 同一个业务含义。
3. 同一个组件样式。
4. 同一套前后台展示链路。
5. 同一套电脑端 / 手机端体验。
6. 澳门 / 香港双系统同类页面结构。

同步只覆盖本次影响范围，不借同步名义重构无关页面。

## 6. 安装初始同步

安装初始采用触发式同步。只有以下情况必须同步：

1. 新增后台菜单或权限。
2. 新增系统配置或默认数据。
3. 新增安装 SQL 或初始化脚本。
4. 新增积分配置、充值入口、帖子资料购买入口。
5. 新增页面路由或默认导航。
6. 新增数据库表或字段并经用户确认。

普通 CSS、普通 UI、小范围文案和样式修复，不强制同步安装初始。

## 7. 技术边界

默认技术栈：

1. PHP 7.3.4+
2. MySQL 5.7+/8.0+ 或 MariaDB
3. 原生 JavaScript + Fetch API
4. HTML5
5. CSS3

不得擅自引入 PHP 框架、Composer 包、前端框架、UI 库、构建工具或模板引擎。
