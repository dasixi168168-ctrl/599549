# 第三轮低风险性能修复：降低客服/会员轮询频率

## 修复原因

排查发现 public/assets/app.js 中客服未读、客服会话、会员状态相关接口轮询频率过高，部分接口 1.5 秒到 3.5 秒请求一次。访客和会员数量增加后，会放大 /public/api.php 压力，造成手机端卡顿、接口变慢、日志增长。

## 本次处理

- 降低客服未读轮询频率。
- 降低客服会话轮询频率。
- 页面隐藏时暂停或降低轮询。
- 接口失败后降低重试频率。
- 避免重复启动多个轮询定时器。

- 没有修改 PHP
- 没有修改 CSS
- 没有修改数据库
- 没有修改接口地址
- 没有修改接口参数
- 没有修改接口返回结构
- 没有修改权限
- 没有修改登录
- 没有修改开奖逻辑
- 没有修改前台页面结构
- 没有修改后台页面
- 没有修改 home-legacy.js
- 没有清缓存
- 没有清日志
- 没有重启服务

## 验证方式

- 检查 public/assets/app.js 语法。
- 检查接口地址和参数没有变化。
- 打开首页，确认页面正常。
- 打开客服入口，确认客服功能仍能轮询更新。
- 切换浏览器到后台，确认不会继续高频轮询。
- 观察访问日志中 /public/api.php 高频请求是否下降。
