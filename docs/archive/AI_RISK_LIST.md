# AI 自动纠偏风险清单

> 本文件列出当前不能随便改的业务点和混乱点。后续修改必须先对照本清单。

## 不可破坏清单

| 类型 | 不可破坏项 |
|---|---|
| 页面 URL | `/`、`/index.php`、`/index.html`、`/404.html`、`/public/index.php`、`/public/record.php`、`/public/forecast.php`、`/public/history.php`、`/public/member.php`、`/public/post.php`、`/public/service.php`、`/public/admin.php` |
| API 契约 | `public/api.php` 的 POST、`action`、请求参数、返回字段、状态码、错误格式 |
| 数据库 | `database/schema.sql` 中已有表、字段、索引、外键、旧数据 |
| 登录 | 前台登录、后台登录、客服坐席登录、session 逻辑 |
| 权限 | 后台 `requireAdminPortal`、后台权限码、角色菜单、危险操作权限 |
| 澳门/香港 | `macau`、`hongkong`、`all` 不能混查、混写、混展示 |
| 业务边界 | 帖子、开奖预测、客服、后台运营、上传记录不能串业务 |
| 上传路径 | `public/uploads/*`、`storage/uploads`、历史图片/附件 URL |
| 后台流程 | 审核、封禁、删除、批量操作、日志、缓存清理、设置保存 |
| 定时任务 | `cli/run_post_generator_schedule.php` |
| 配置 | `config/database.php`、`.user.ini`、`.htaccess`、服务器配置 |

## 混乱点分类

| 类型 | 当前表现 | 风险等级 | 处理建议 |
|---|---|---|---|
| 目录结构 | 根目录混有入口、静态页、部署包、公开上传和运行目录 | 高 | 先标记用途，不移动不删除 |
| 前台页面 | 澳门/香港首页是两套入口，部分双地区页面靠 `region` 参数 | 高 | 不改 URL，先补清单 |
| 后台页面 | `public/admin.php` 单文件承载登录、页面分发和表单处理 | 高 | 先只读梳理，暂不拆分 |
| API 接口 | `public/api.php` 单文件 action 很多，前台/后台/客服混在一起 | 高 | 不改契约，后续只做清单和测试 |
| 数据库结构 | 前台权限和后台权限两套表并存，表较多 | 高 | 不删字段，不迁移核心表 |
| 权限逻辑 | 后台权限存在，但区域/业务数据权限需逐模块确认 | 极高 | 暂不放宽权限 |
| 地区逻辑 | `region` 分散在入口、SQL、模板和样式中 | 高 | 不合并澳门/香港查询 |
| 业务边界 | 未见明确业务 A/B 命名，当前按模块边界运行 | 高 | 需业务确认后再命名 |
| UI 样式 | `public/styles/style.css` 体量大，前后台样式需分开评估 | 中 | 只做单点小修 |
| 性能 | 后台列表、帖子、日志、客服轮询需确认分页和索引 | 中高 | 先查高频页面 |
| 安全 | 配置敏感、上传公开目录、安装入口、后台入口需重点保护 | 高 | 先检查暴露面，不改权限 |
| 合规 | 含六合彩/预测相关功能 | 高 | 澳门/香港上线前需本地合规核验 |

## 暂时不能动的高风险模块

| 模块 | 原因 |
|---|---|
| 登录/注册 | 影响用户会话和访问入口 |
| 后台权限系统 | 误改会造成越权或后台不可用 |
| 核心 API | 影响前台、后台、客服、移动端调用 |
| 数据库核心表 | 影响旧数据和旧查询 |
| 澳门/香港数据过滤 | 误改会串地区 |
| 帖子购买/积分 | 涉及用户资产和付费内容 |
| 上传路径 | 影响历史图片、附件和富文本内容 |
| 客服会话 | 涉及用户沟通记录 |
| 定时任务 | 可能影响自动发帖/更新 |
| 安装和配置 | 影响生产可用性和安全 |
