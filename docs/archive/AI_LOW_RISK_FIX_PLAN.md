> 本文件只定义后续小步纠偏顺序，不改变任何业务代码。

## 推荐后续低风险顺序

| 顺序 | 模块 | 最小动作 | 不可动点 | 测试 |
|---|---|---|---|---|
| 1 | 架构边界纠偏清单 | 对入口、API action、Service、模板、CSS/JS 职责做只读归类，标记可整理点和不可动点 | 不拆入口，不改旧 API，不改登录权限 | 对照 `public/*.php`、`app/Services`、`app/Views`、`public/assets` |
| 2 | 客服状态矩阵 | 建立 `guest/member/agent/admin`、有消息/无消息、在线/离线、macau/hongkong、电脑/手机状态清单 | 不改客服业务、登录态、轮询 action、会话绑定 | 访问 `public/service.php` 和后台客服页，逐状态核对结构 |
| 3 | API 清单测试脚本 | 新增只读/手工测试清单，不调用生产写接口 | 不改 API action、参数、返回字段、状态码 | 对照旧参数和返回格式 |
| 4 | 手机端轻量样式 | 只修明确横向溢出、按钮不可点或同组件状态错位问题 | 不改布局结构，不改 JS 行为 | 手机宽度访问核心页面 |
| 5 | 后台列表可读性 | 只处理表格显示、空状态、筛选提示 | 不改查询条件、分页、权限 | 后台列表页访问测试 |
| 6 | 上传路径风险清单 | 只记录公开上传目录和引用位置 | 不移动、不删除上传文件 | 打开历史图片/附件 URL |
| 7 | SEO 标题描述检查 | 检查首页、历史、预测、会员页面标题和描述是否地区匹配 | 不改 URL，不改跳转 | 查看页面 title/description |
| 8 | 文案/本地化检查 | 只在架构和状态一致性问题稳定后，记录明显地区文案问题 | 不把纠偏方向改成文案优化 | 澳门/香港首页、会员页、客服页 |

## 客服状态矩阵优先检查项

| 维度 | 状态 |
|---|---|
| 登录态 | 未登录、会员已登录、客服坐席已登录、后台管理员已登录 |
| 地区 | `macau`、`hongkong` |
| 数据 | 有聊天记录、无聊天记录、有会话、无会话 |
| 在线状态 | 客服在线、客服离线、对方输入中、图片发送中、语音发送中 |
| 终端 | 电脑端、手机端 |
| 异常 | 加载中、轮询失败、无权限、会话关闭、会话屏蔽 |

## 低风险结构纠偏边界

| 项目 | 要求 |
|---|---|
| 入口 | `public/index.php`、`public/admin.php`、`public/api.php`、`public/service.php` 暂不拆分 |
| API | 保持 `action` 分发、请求参数、返回字段、状态码、错误格式不变 |
| 业务 | 帖子、开奖预测、客服、后台运营、上传记录不串模块 |
| 地区 | `macau`、`hongkong`、`all` 不混查、不混写、不混展示 |
| UI | 同一组件跨登录态、数据态、地区、双端应复用结构和基础 class |
| 性能 | 列表、日志、客服轮询、上传记录优先检查分页、索引和缓存风险 |
| 安全 | 不放宽登录、权限、CSRF、Session、上传限制和客服会话绑定 |

## 每次低风险修改前必须确认

| 检查项 | 要求 |
|---|---|
| 是否影响旧 URL | 影响则暂停 |
| 是否影响旧 API | 不允许改变请求参数、返回字段、状态码 |
| 是否影响数据库 | 不允许删除、改名、改主表结构 |
| 是否影响登录/权限 | 不允许放宽权限 |
| 是否影响澳门/香港 | 必须证明数据不串 |
| 是否影响业务边界 | 必须证明模块数据不串 |
| 是否影响上传 | 历史路径必须继续可用 |
| 是否可回滚 | 必须有明确恢复步骤 |

## 下一个建议执行项

建议下一个执行项为“架构边界纠偏清单”，先只读归类 `public/*.php`、`app/Services`、`app/Views`、`public/assets`、`public/styles` 的真实职责和重复点，再选择一个不影响业务的小范围结构或 UI 状态一致性修复。
