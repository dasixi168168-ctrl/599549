# 预测规则能力矩阵

更新时间：2026-07-10

本文档只固化当前代码已经具备的预测规则能力边界，不代表命中率承诺，不替代真实回测结果。

## 依据文件

- `app/Services/PredictionService.php`
- `app/Services/ForecastAdvancedAnalysisService.php`
- `config/forecast_advanced_rules.json`
- `public/api.php`
- `public/forecast.php`
- `tools/advanced_analysis_boundary_backtest.php`

## 状态定义

| 状态 | 含义 |
| --- | --- |
| 已支持 | 现有代码已有明确实现，并已接入预测主流程或高级分析输出。 |
| 部分支持 | 已有基础能力或配置入口，但还缺少完整闭环、显式配置、训练流程或验收记录。 |
| 未支持 | 当前代码未看到对应实现，不能按已具备能力对外声明。 |

| --- | --- | --- | --- |

## 算法能力

| 能力项 | 状态 | 当前实现 | 差距 / 说明 |
| --- | --- | --- | --- |
| 频率统计 | 已支持 | 统计全码、正码、特码、生肖、波色、五行、单双、大小、头数、尾数。 | 无明显缺口。 |
| 热度等级 | 已支持 | 号码等级、正码等级、特码等级、生肖等级、正码生肖等级、特码生肖等级。 | 无明显缺口。 |
| 冷热号识别 | 已支持 | `cold_numbers`、`active_numbers`、核心正码、温正码、热特码、温特码。 | 无明显缺口。 |
| 规则引擎筛选 | 已支持 | 支持生肖类型、号码类型、平特类型、其他类型筛选；多筛选先取交集，交集为空时合并。 | 无明显缺口。 |
| 推荐号生成 | 已支持 | 优先筛选命中热号池，再全热号池，最后 1-49 兜底。 | 无明显缺口。 |
| 组合生成 | 已支持 | 生成 6 正码 + 1 特码组合，检查单双、头尾、生肖、波色、冷号等规则。 | 组合数量当前最多 3 组。 |
| 重复度控制 | 已支持 | 推荐号码和展示 payload 与上一预测结果做重复度限制。 | 无明显缺口。 |
| 稳定置信度扰动 | 已支持 | 使用地区、期号、号码、筛选等生成稳定 jitter，不是每次随机飘动。 | 无明显缺口。 |
| 外部规则权重配置 | 已支持 | `config/forecast_advanced_rules.json`，并支持 JSON/YAML 覆盖默认规则。 | 当前主要覆盖 advanced_analysis。 |
| 可扩展机器学习权重 | 部分支持 | 已有可配置权重入口，可作为未来模型权重接入点。 | 未看到 ML 训练、模型加载、特征向量、模型版本、推理器或评估流水线。不能声明已支持机器学习。 |
| 自动学习 / 在线训练 | 未支持 | 未看到基于历史命中率自动更新权重的流程。 | 如要支持，需要新增训练/评估/发布闭环。 |
| 模型版本管理 | 未支持 | 当前版本为规则版本 `2026-high-priority-1`。 | 没有独立 ML model version。 |

## 当前结论

当前系统可以对外描述为：

- 支持结构化预测 JSON 输出。
- 支持频率统计 + 规则引擎 + 外部可配置规则权重。
- 支持生肖、五行、合数属性、冷热、遗漏、和值、跨度、杀号、胆码拖码等扩展模块。

当前系统不应对外描述为：

- 已具备完整机器学习模型。
- 已具备自动训练或在线学习。
- 已具备显式遗漏阈值触发规则。

## 建议补齐项

| 优先级 | 能力项 | 建议 |
| --- | --- | --- |
| 高 | 遗漏阈值 | 在规则配置中新增 `omission.thresholds`，支持号码、尾数、生肖、五行的显式阈值。 |
| 中 | ML 权重接口 | 先定义离线模型输出格式，如 `config/forecast_ml_weights.json`，不要直接引入新依赖。 |
| 中 | 规则版本说明 | 在预测输出中增加规则配置版本摘要，便于回测对照。 |
| 低 | 文档同步 | 每次调整预测规则后更新本文档，并执行固定边界回测。 |
